### 核心设计

**数据权限注解：**
```java
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface DataScope {
    String deptAlias() default "d";   // 部门表别名
    String userAlias() default "u";   // 用户表别名
    boolean deptRelation() default true; // 是否关联部门表
}
```

**拦截器逻辑：**
- 查当前用户角色 → 取 dataScope 最大值（多角色取最宽）
- scope=1 全部 → 不改写 SQL
- scope=4 本人 → 追加 `AND u.id = #{currentUserId}`
- scope=2 本部门 → 追加 `AND d.id = #{currentDeptId}`
- scope=3 本部门及下属 → 追加 `AND d.id IN (本部门及子孙部门ID)`
- 超级管理员（admin）跳过，看全部

**维保任务新字段：**
```sql
ALTER TABLE maintenance_task ADD COLUMN equipment_id BIGINT COMMENT '故障设备ID' AFTER point_id;
ALTER TABLE maintenance_task ADD COLUMN fault_desc VARCHAR(1000) COMMENT '故障现象' AFTER description;
ALTER TABLE maintenance_task ADD COLUMN solution VARCHAR(1000) COMMENT '处理方法' AFTER fault_desc;
ALTER TABLE maintenance_task ADD COLUMN parts_used VARCHAR(500) COMMENT '更换配件' AFTER solution;
ALTER TABLE maintenance_task ADD COLUMN plan_inspect_date DATE COMMENT '计划巡检日期' AFTER plan_date;
```

**自动巡检生成：**
- 定时任务 `MaintenanceInspectScheduler`：每天 02:00 扫描维保主合同（ACTIVE）+ 巡检计划，按设备生成 INSPECTION 类型工单
- 接口 `POST /business/maintenance-task/generate-inspection?contractId=xxx` 手动触发生成

### 核心设计

**合同收付款表：**
```sql
CREATE TABLE contract_payment (
  id BIGINT PRIMARY KEY,
  code VARCHAR(50) NOT NULL COMMENT '单号',
  contract_id BIGINT NOT NULL COMMENT '合同ID',
  type VARCHAR(20) NOT NULL COMMENT 'RECEIVABLE应收/PAYABLE应付',
  plan_date DATE COMMENT '计划日期',
  plan_amount DECIMAL(14,2) COMMENT '计划金额',
  actual_amount DECIMAL(14,2) COMMENT '实际金额',
  actual_date DATE COMMENT '实际日期',
  invoice_no VARCHAR(100) COMMENT '发票号',
  status VARCHAR(20) DEFAULT 'PENDING' COMMENT 'PENDING/RECEIVED/OVERDUE',
  remark VARCHAR(500),
  create_by BIGINT, create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
  update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP, deleted TINYINT DEFAULT 0
);
```

**审批流模型（轻量）：**
```sql
-- 审批流定义（按业务类型：CONTRACT_APPROVAL/QUOTE_APPROVAL）
approval_flow(id, code, name, business_type, enabled)
-- 审批节点（线性流转，支持金额分级条件）
approval_node(id, flow_id, node_order, node_name, approver_role_id, amount_threshold)
-- 审批记录（每次审批留痕）
approval_log(id, flow_id, business_type, business_id, node_order, approver_id, result, opinion, approve_time)
```
- 合同/报价「发起审批」→ 按 business_type 找 enabled 流程 → 创建首个节点 → 审批人同意则推进下一节点 → 全通过则 approvalStatus=APPROVED
- 金额分级：节点可配 amount_threshold，合同金额 ≥ 阈值才走该节点
