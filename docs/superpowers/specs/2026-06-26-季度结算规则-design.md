# 季度结算规则 设计规格

- **版本**: v1
- **日期**: 2026-06-26
- **状态**: 已确认,待落地
- **关联文档**: [需求文档.md](../../../需求文档.md) 第 3.5.5 节

---

## 一、背景

维护型项目的维保主合同采用"按季度结算"的周期性付费模式。由于合同期不固定、起止时间随机、客户财务不强制自然季度对账,需要对"季度"的精确定义、结算单生成时点、金额计算、变更与终止处理做明确约束,避免开发实现时产生歧义。

## 二、核心决策

- **季度口径**: 合同季度(从合同生效日起算,每 3 个月一期)
- **合同期约束**: 3 个月的整数倍,最短半年
- **结算单生成**: 期末次日自动生成草稿
- **金额变更**: 下期生效,当期不补差
- **提前终止**: 线下协商,系统提供手动调整入口(需审批留痕)

---

## 三、详细规则

### 规则 1: 合同期约束

| 项 | 约束 |
|----|------|
| 单位 | 3 个月(一个季度) |
| 最短 | 半年(2 期) |
| 允许长度 | 3 个月的任意整数倍(半年 / 9 个月 / 1 年 / 15 个月 / 2 年 ...) |
| 系统校验 | `合同期月数 % 3 == 0 且 ≥ 6`,否则保存失败并提示"合同期必须为 3 个月的整数倍且不少于半年" |

### 规则 2: 季度计算(合同季度)

- **起算点**: 合同生效日
- **每期长度**: 3 个月
- **计算公式**:
  - 第 N 期开始日 = `生效日 + (N-1) × 3 个月`
  - 第 N 期结束日 = `生效日 + N × 3 个月 - 1 天`
- **合同到期日** = `生效日 + 合同期 - 1 天`

**示例**(生效日 2024-03-15,合同期 1 年):

| 期次 | 开始日 | 结束日 |
|------|--------|--------|
| Q1 | 2024-03-15 | 2024-06-14 |
| Q2 | 2024-06-15 | 2024-09-14 |
| Q3 | 2024-09-15 | 2024-12-14 |
| Q4 | 2024-12-15 | 2025-03-14 |

合同到期日 = 2025-03-14

### 规则 3: 结算单生成

| 项 | 规则 |
|----|------|
| 生成时点 | 每期结束日的次日 00:00 自动生成(如 Q1 结束日 06-14,则 06-15 生成) |
| 生成方式 | 定时任务自动生成草稿单 |
| 单据内容 | 本期维保费金额、本期巡检次数、本期维保工单汇总、关联维保记录 |
| 状态流转 | 草稿 → 审核 → 确认 → 开票 → 回款 → 关闭 |
| 通知 | 生成时同时站内消息通知项目经理和财务 |

### 规则 4: 金额计算

- **单期金额** = `合同总额 ÷ 期数`
- **尾差处理**: 前 N-1 期保留 2 位小数,最后一期 = `合同总额 - 前 N-1 期之和`(吸纳尾差)
- **说明**: 合同期严格为 3 个月整数倍,**不存在按天折算**场景,金额整齐

**示例**(合同总额 10000 元,4 期):
- Q1/Q2/Q3 = 2500.00 元
- Q4 = 10000 - 7500 = 2500.00 元

**示例**(合同总额 10000 元,3 期):
- Q1/Q2 = 3333.33 元
- Q3 = 10000 - 6666.66 = 3333.34 元(吸纳尾差)

### 规则 5: 金额变更

| 项 | 规则 |
|----|------|
| 触发 | 合同变更流程(补充协议上传 + 审批通过) |
| 生效时点 | 从下一期开始按新金额结算 |
| 当期处理 | 仍按旧金额结算,不做按天补差 |
| 留痕 | 结算单显示"适用金额版本",变更历史可追溯 |

### 规则 6: 提前终止

- 系统不提供自动解约结算逻辑
- 由商务线下协商后,通过"手动调整结算单"功能处理
- 调整操作需财务审批,必须填写调整原因
- 全程留痕,纳入审计日志
