# 验收规则 设计规格

- **版本**: v1
- **日期**: 2026-06-26
- **状态**: 已确认,待落地
- **关联文档**: [需求文档.md](../../../需求文档.md) 第 3.5.8 节

---

## 一、背景

维护型项目的点位报价费采用"验收后结算"模式。验收环节涉及验收人、发起时点、结论时效、整改上限、验收依据等关键约束,若不明确会导致结算流程卡死或被无限拖延。本规格定义完整的验收规则,作为开发实现的明确依据。

## 二、核心决策

- **验收人**: 项目经理单签
- **发起时点**: 施工完成即自动生成草稿
- **结论时效**: 无强制时效,每周催办一次
- **整改上限**: 3 次,超出升级管理层仲裁
- **系统使用范围**: 仅内部使用,客户不登录系统

---

## 三、详细规则

### 规则 1: 验收人
- 项目经理单签即生效,无需客户或第三方签字
- **风险提示**: 项目经理单签存在客户后续不认可的风险,通过规则 5 的"选填客户确认附件"缓解

### 规则 2: 发起时点
- 施工状态置为"已完成"时,系统**自动生成验收单草稿**
- 项目经理在草稿上填写实际工程量和验收结论

### 规则 3: 结论时效
- **无时效约束**, 项目经理可随时出结论
- 系统**每周催办一次**(站内消息),提醒有未决验收单
- 不强制升级, 不视同通过

### 规则 4: 整改上限
- 验收不通过 → 进入"整改中"状态 → 整改完成后重新验收
- 整改次数上限 **3 次**
- 超过 3 次仍未通过, **强制升级到管理层仲裁**
- 仲裁结论: 继续整改 / 终止该点位 / 特殊处理(需填写仲裁说明)

### 规则 5: 验收单字段
- **必填**: 实际工程量(与报价明细对比)
- **选填**: 客户确认附件(线下获取客户签字后上传,降低扯皮风险)
- **系统自动填充**: 验收单编号、关联点位、关联报价、生成时间、项目经理

### 规则 6: 验收与结算的关系
- 验收通过 → 该点位报价金额进入结算队列
- 验收不通过 → 不可结算(系统强校验,R3 规则)
- 整改中 → 不可结算
- 仲裁中 → 不可结算,待仲裁结论后按结论处理

### 规则 7: 系统使用范围约束 ⭐
- **本系统仅内部使用,客户不会登录系统**
- 客户相关动作(确认、签字、付款承诺)均通过线下完成,内部人员上传附件留痕
- 系统不提供客户账号、客户端、客户门户

## 四、验收状态流转

```
[点位施工中]
     │
     ▼ 施工完成
[验收单草稿] ──(自动生成)
     │
     ▼ 项目经理填写结论
     ├─→ 通过 ────────→ [已验收] ─→ 进入结算队列
     │
     ├─→ 不通过 ──→ [整改中] ─→ 整改完成 ─→ 重新验收(回到草稿)
     │                    │
     │                    └─(整改次数 > 3)─→ [仲裁中] ─→ 仲裁结论
     │                                                    │
     │                                                    ├─→ 继续整改
     │                                                    ├─→ 终止点位
     │                                                    └─→ 特殊处理
     │
     └─→ (每周催办,无强制升级)
```
