# 智能化工程管理系统

面向智能化工程行业的一体化管理平台，覆盖项目合同、报价、进度、验收、维保全生命周期。系统针对工程公司长期存在的合同分散、报价难追溯、进度不透明、维保脱节等痛点，提供从商机报价到维保结算的闭环数字化管理能力。

本项目全程通过 **TRAE AI 编程助手** 自动化完成开发，包括需求分析、架构设计、代码实现、数据库建模、接口联调与前端页面构建，无需人工编写业务代码。详见文末[开发过程说明](#开发过程说明)。

## 核心功能

系统围绕两条业务主线展开：

- **新建工程线**：项目立项 → 合同签订 → 报价管理 → 设备台账 → 进度跟踪 → 项目验收 → 维保服务
- **维护型项目线**：维护型项目立项 → 维保主合同（季度结算） + 多点位报价 → 点位施工 → 点位验收 → 报价费结算（验收后结算）

两条线共享报价、进度、验收等基础能力，通过 `business_type` 字段区分业务来源，避免代码重复。

## 快速开始

### 环境要求

- JDK 17+
- Maven 3.8+
- Node.js 18+ / pnpm
- MySQL 8.x
- Redis 7.x

### 后端启动

```bash
cd server
# 修改 src/main/resources/application-dev.yml 中的数据库和 Redis 连接地址
mvn spring-boot:run
```

后端启动在 `http://localhost:8080/api`，首次启动时 `DbInitRunner` 会自动幂等创建所有业务表并写入种子数据（管理员账号 `admin` / `admin123`）。

### 前端启动

```bash
cd web
pnpm install
pnpm dev
```

前端启动在 `http://localhost:5173`，所有 `/api` 请求自动代理到后端 8080 端口。浏览器打开后使用 `admin` / `admin123` 登录。

## 关键设计决策

**单租户架构**：系统仅内部使用，不加 `tenant_id`，通过 RBAC + 部门级数据权限隔离。简单直接，未来需要多租户时新部署一套即可。

**基础能力层 + 业务编排层**：报价、进度、验收作为基础能力独立存在，维护型项目模块通过编排调用这些能力，以 `business_type` 字段区分业务来源，避免实体重复定义。

**合同季度结算**：以合同生效日为起算点，每 3 个月一期。期末次日由定时任务自动生成草稿结算单。单期金额 = 合同总额 / 期数，除不尽时尾差归入最后一期。金额变更经审批后从下一期生效，当期不补差。

**验收规则**：项目经理单签生效，无需客户签字（客户确认线下获取后上传附件留痕）。整改上限 3 次，超出升级管理层仲裁。验收通过后方可发起结算，系统强校验。

**Long ID 序列化为 String**：后端所有 Long 类型 ID 经 JacksonConfig 统一序列化为 String，避免前端 JavaScript 精度丢失。前端所有关联实体 ID 字段使用 `el-select` 下拉选择，禁止文本输入。

**数据权限**：通过 `@DataScope` 注解 + MyBatis 拦截器实现，按角色配置数据范围（1 本人 / 2 本部门 / 3 本部门及下属 / 4 全部），超级管理员跳过过滤。

## 定时任务

| 任务 | 执行时间 | 功能 |
|------|---------|------|
| 季度结算单生成 | 每日 00:00 | 扫描有效维保合同，幂等生成到期季度的结算草稿单 |
| 巡检工单生成 | 每日 02:00 | 扫描有效合同下点位，生成当月巡检工单 |

## 开发过程说明

本项目全程通过 **TRAE AI 编程助手** 自动化完成开发，涵盖从需求分析到代码实现的全流程，无人工编写业务代码。

### 开发方法论

项目采用 Superpowers 技能框架驱动，核心方法论包括：

- **设计先于编码**：每个开发阶段动手前先产出设计规格文档（租户架构、模块分层、结算规则、验收规则、UI 规范），解决争议点后再写代码
- **子代理驱动开发**：每个开发阶段分配独立子代理执行，阶段间设置审查检查点，确保质量可控
- **计划驱动跟踪**：每份实现计划用任务复选框逐项跟踪进度，每个任务包含文件结构、代码片段、验证标准
- **验证先于完成**：每个阶段结束前必须通过编译、启动、接口验证、前端可操作四项检查，确认通过后才标记完成
